<h1>Kufar Telegram Notifier</h1>
<b>Kufar Telegram Notifier</b> - бот, который раз в заданное время ведет опрос официального Kufar API для поиска новых объявлений по заданным ключевым словам для последующей отправки результатов в Telegram чат.<br>
Использование данного бота предусматривается только для ознакомительных целей.<br><br>
<pre>Если Вы являетесь администрацией Kufar и у Вас имеются претензии к данному проекту - обращайтесь в раздел «<i>Issues</i>» для обратной связи.</pre>

## Развёртывание на Railway

В репозитории есть `Dockerfile` и `railway.json`, поэтому Railway автоматически соберёт и запустит бота.

1. Создайте сервис из этого GitHub-репозитория.
2. Добавьте переменные сервиса:
   - `TELEGRAM_BOT_TOKEN` — токен, выданный BotFather.
   - `TELEGRAM_CHAT_ID` — ID личного чата или группы, куда бот будет отправлять объявления.
   - `KUFAR_QUERIES_JSON` — JSON-массив поисковых запросов из раздела `queries`.
   - `KUFAR_RECIPIENTS_JSON` — необязательный JSON-массив получателей с отдельными запросами; если задан, заменяет `KUFAR_QUERIES_JSON`.
3. Создайте Railway Volume, подключите его к сервису и укажите mount path `/data`.
4. Оставьте одну реплику сервиса: несколько реплик с одним кешем могут отправлять повторные уведомления.

Если `TELEGRAM_CHAT_ID` ещё неизвестен, сначала добавьте только `TELEGRAM_BOT_TOKEN`, отправьте `/start` своему боту и запустите деплой. В логах появится строка `[SETUP]: TELEGRAM_CHAT_ID=...`. Добавьте найденное значение в переменные и повторите деплой.

При первом запуске бот сам создаст `/data/cached-data.json`. Файл остаётся на Volume после перезапусков и новых деплоев, поэтому уже обработанные объявления не отправляются повторно.

Поисковые запросы задаются через `KUFAR_QUERIES_JSON`, чтобы менять фильтры без нового коммита и деплоя. Например: `[{"tag":"iPhone","only-title-search":true,"limit":10}]`.

Для нескольких получателей используйте `KUFAR_RECIPIENTS_JSON`. У каждого получателя свой кеш, поэтому одинаковое объявление не дублируется при пересечении его запросов: `[{"chat-id":123456789,"queries":[{"tag":"iPhone","only-title-search":true,"limit":10}]}]`.

При существенном изменении фильтров существующего запроса задайте ему новое значение `cache-key`. Тогда бот сначала без уведомлений запомнит текущие результаты изменённого запроса и не пришлёт старые объявления как новые.

Объявления из категории Kufar «Спрос» исключаются для всех получателей. Для отдельного запроса можно исключать заголовки через `exclude-title-groups`: каждая внутренняя группа срабатывает, только если в заголовке найдены все её части. Например, `"exclude-title-groups":[["гараж","пролет"]]` исключает гаражные пролёты только из того запроса, где задано это правило.

Если поисковая система игнорирует короткое слово, можно потребовать точную фразу в заголовке через `required-title-phrases`. Например, `"required-title-phrases":["не включается"]` пропустит только объявления, в заголовке которых действительно есть эта фраза.

Бот также перепроверяет многословные запросы после получения результатов от Kufar. В заголовке должны присутствовать все слова запроса, поэтому `не проверял` больше не совпадает с объявлением, где есть только `проверял` или только `не`. Для запросов, начинающихся с `не`, дополнительно сохраняется точный порядок слов, чтобы не потерять смысл отрицания. Однословные запросы работают как раньше.

Задержки можно переопределить переменными `KUFAR_QUERY_DELAY_SECONDS` и `KUFAR_LOOP_DELAY_SECONDS`. Первая задаёт паузу между отдельными запросами к Kufar, вторая — целевой интервал между началами полных циклов. Для проверки примерно раз в пять минут используйте `KUFAR_LOOP_DELAY_SECONDS=300`. Значения Telegram из файла конфигурации служат только шаблоном и переопределяются переменными Railway.

Бот хранит на Volume не только идентификаторы объявлений, но и последнюю замеченную и минимальную цены. Уведомление приходит только при установлении новой минимальной цены, поэтому обычное снижение после искусственного повышения не считается выгодной скидкой. В сообщении показываются старая цена, новая цена, сумма и процент выгоды. Нулевая или договорная цена не создаёт ложное уведомление. При обновлении со старого формата кэша текущие цены становятся безопасной начальной точкой без рассылки старых скидок.

Команда `/status` доступна всем получателям из `KUFAR_RECIPIENTS_JSON` и показывает количество активных запросов, время последней успешной проверки и размер кэша этого получателя. Сообщения из посторонних чатов игнорируются.

Команда `/menu` (также `/start`) открывает простое меню с кнопками:

- `📋 Мои запросы` — показывает личный список, объединяя одинаковые запросы и перечисляя их категории в одной строке;
- `➕ Добавить запрос` — спрашивает текст, позволяет отметить галочками одну или несколько категорий, при необходимости включить режим `Только частные продавцы`, а затем добавить всё одной кнопкой;
- `➖ Удалить запрос` — показывает каждый текст запроса один раз и после подтверждения удаляет все его категории;
- `ℹ️ Статус` — тот же безопасный статус, что и `/status`;
- `❓ Помощь` — коротко объясняет весь процесс добавления запроса.

Чтобы большое количество вариантов не перегружало экран, выбор категорий разделён на две страницы. На первой находятся самые востребованные категории и автоматический подбор, а кнопка `📚 Ещё категории` открывает компьютеры, телефоны, мебель, сад, детские товары, животных, красоту и здоровье и другие дополнительные разделы. Выбранные галочки сохраняются при переходе между страницами.

Изменения, сделанные через Telegram, сохраняются в `query-overrides` внутри файла кэша на Railway Volume. Они переживают перезапуск и не удаляют кэш просмотренных объявлений. При добавлении нового запроса найденные на первом проходе объявления только запоминаются — уведомления начинаются со следующих новых объявлений.
